SSH Security
How SSH Tunnels Work
1. Agent connects to XTunnel SSH Gateway (port 2222)
2. TLS 1.3 encrypts all traffic (RSA-4096 host key)
3. JWT token authenticates the agent
4. HMAC-SHA256 signs all forwarded requests
5. SSH traffic is bridged to your local SSH server
→ Your SSH server never exposes a public port
Connect via SSH Tunnel
1. Start tunnel: xtunnel ssh 22
2. Note the TCP port (e.g. 10042)
3. Connect: ssh -p 10042 user@xtunnel.io
✓ Works through firewalls and NAT
✓ TLS 1.3 + JWT authentication
✓ IP allowlist support
Security Configuration
#Start SSH tunnel with IP allowlist
$xtunnel ssh 22 --token $XTUNNEL_TOKEN
 
#Connect from outside (uses public TCP port)
$ssh -p <port> user@xtunnel.io
 
#Use SSH key (recommended) instead of password
$ssh -i ~/.ssh/id_ed25519 -p <port> user@xtunnel.io
